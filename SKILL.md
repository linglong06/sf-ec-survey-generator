description: SAP SuccessFactors Employee Central (EC) 项目调研问卷生成器。用于SF项目蓝图设计阶段，自动生成结构化的Excel调研问卷。支持从客户制度文档中提取答案预填充问卷，支持中英文双语输出，覆盖组织管理、岗位管理、人事管理等EC核心模块。触发场景：(1) 用户需要生成SF调研问卷 (2) 用户提到"SF问卷"、"EC调研"、"SuccessFactors问卷"、"人事系统调研" (3) 用户需要进行SF项目需求收集 (4) 用户需要蓝图设计阶段的调研文档 (5) 用户提供公司制度、员工手册、人力资源管理制度等文档需要生成调研问卷。

# SF EC Survey Generator

生成 SAP SuccessFactors Employee Central 蓝图设计阶段调研问卷的专用工具。

## 工作流程

### 完整流程（推荐）

当用户提供制度文档时，按以下流程操作：

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  1. 展示问题    │ ──▶ │  2. 用户确认    │ ──▶ │  3. 提取答案    │ ──▶ │  4. 生成问卷    │
└─────────────────┘     └─────────────────┘     └─────────────────┘     └─────────────────┘
  展示完整问题列表        用户确认或调整问题      Claude分析文档          按确认的问题生成Excel
  按类别分组展示          生成临时问题列表        提取SF EC相关信息
        │                       │                      │                      │
        │                       │                      │                      │
        └───────────────────────┴──────────────────────┴──────────────────────┘
                           ⚠️ 问题调整仅影响本次问卷，不修改问题库文件
```

**Step 1: 展示问题列表（必须步骤）**

⚠️ **重要：在生成问卷前，必须先向用户展示完整问题列表**

Claude会按以下格式展示问题列表，让用户确认：

```
📋 SF EC 调研问卷问题库（共90题）

════════════════════════════════════════
1. 基本信息 (10题)
════════════════════════════════════════
  1.1 公司概况 (5题)
    1. 请简要介绍公司的业务范围和主营业务？
    2. 公司目前的员工总数是多少？
    ...

  1.2 现有系统 (5题)
    6. 目前使用的人事管理系统是什么？
    ...

════════════════════════════════════════
2. 组织管理 (12题)
════════════════════════════════════════
...

请确认：
1. 是否需要调整问题范围？（如只生成特定模块）
2. 是否需要修改或补充问题？
3. 确认无误后回复"继续"生成问卷。
```

**Step 2: 用户确认与问题调整**

用户可以：

- ✅ **确认**：回复"继续"或"确认"，进入下一步
- ✏️ **修改问题**：指出需要修改的问题内容，Claude调整后再次确认
- ➕ **添加问题**：提出需要新增的问题，Claude添加到当前问卷
- ➖ **删除问题**：指定不需要的问题，Claude从当前问卷移除
- 📦 **精简模块**：指定只生成特定模块（如"只生成组织管理和人事管理"）

⚠️ **重要规则：问题调整仅在当前会话生效**

- 用户在对话中临时调整的问题**只影响本次问卷生成**
- Claude **不会修改** `references/ec_questions.md` 或 `scripts/generate_survey.py` 中的问题库
- Claude 会在当前会话中维护一个**临时问题列表**
- 最终问卷按照**用户确认后的临时问题列表**生成
- 问题库文件保持不变，下次对话仍可使用完整问题库

**Step 3: 收集客户制度文档**

请用户提供以下文档（至少提供一种）：

- 公司制度汇编
- 员工手册
- 人力资源管理制度
- 组织架构说明文档
- 岗位职责说明书
- 薪酬管理制度
- 考勤管理制度
- 劳动合同管理规范

**Step 4: Claude从文档中提取答案**

Claude会阅读并分析文档，提取与SF EC模块相关的答案。

**重要规则：**

1. **答案必须来源于用户提供的文档**，严禁在文档之外自行编造或补充答案
2. **记录答案来源**：包括源文件名称和原文章节/段落
3. **找不到答案时**：在答案列填写"未在文档中找到相关信息"
4. **信息不完整时**：提取现有信息，标注"需补充"
5. **多个文档冲突时**：提取所有版本，标注"存在冲突，需确认"

关键信息提取方向：

- **组织管理**: 组织架构类型、层级定义、编码规则、审批流程
- **岗位管理**: 岗位体系、职级体系、编制管理规则
- **人事管理**: 入职流程、转正规则、调动流程、离职流程、合同管理
- **薪酬考勤**: 薪酬结构、工时制度、加班规则
- **数据权限**: 权限控制规则、敏感数据定义

**Step 5: 生成带答案的问卷**

使用用户确认后的**临时问题列表**生成问卷：

```bash
python3 scripts/generate_survey.py -o ./问卷.xlsx -a answers.json
```

⚠️ **重要说明**：
- 问卷内容基于**用户确认后的临时问题列表**，而非原始问题库
- 如果用户在对话中对问题进行了调整（修改、添加、删除），问卷将反映这些调整
- 问题库文件 (`references/ec_questions.md`, `scripts/generate_survey.py`) 保持不变

### 快速生成（无文档时）

如果没有客户制度文档，可直接生成空白问卷：

```bash
python3 scripts/generate_survey.py -o ./survey.xlsx
```

## 命令参数


| 参数                   | 说明                  | 默认值                   |
| -------------------- | ------------------- | --------------------- |
| `-o, --output`       | 输出文件路径              | `./sf_ec_survey.xlsx` |
| `-c, --categories`   | 指定类别（可多选）           | 全部类别                  |
| `-l, --language`     | 语言：`cn`/`en`/`both` | `both`                |
| `-a, --answers`      | 答案JSON文件路径          | -                     |
| `--no-answer-column` | 不包含答案列              | 包含                    |
| `--no-source-column` | 不包含来源列              | 包含                    |
| `--no-sub-category`  | 不包含子类别行             | 包含                    |
| `--template`         | 生成答案JSON模板          | -                     |
| `--list-categories`  | 列出所有类别              | -                     |


### 可用类别

- `基本信息` - 公司概况、现有系统
- `组织管理` - 组织架构、组织层级、组织审批
- `组织信息` - 组织基础信息、组织属性
- `岗位管理` - 岗位体系、岗位信息、岗位编制
- `人事管理` - 员工信息、入职、转正、调动、离职、合同、薪酬、考勤、权限、报表

## 使用示例

### 场景1：从制度文档生成问卷

用户提供员工手册后：

```
1. Claude展示问题列表，用户确认
2. Claude阅读员工手册
3. 提取与SF EC相关的答案，记录来源
```


### 场景3：仅中文版本

```bash
python3 scripts/generate_survey.py -o ./survey.xlsx -l cn
```

## 答案JSON格式

```json
{
  "_说明": "这是答案模板文件",
  "_格式": "问题中文内容: {answer: 答案内容, source: 来源信息}",
  "公司目前的员工总数是多少？": {
    "answer": "约5000人",
    "source": "员工手册.pdf / 第一章 公司概况"
  },
  "试用期转正的业务规则是什么？": {
    "answer": "试用期一般为3-6个月，转正需通过部门考核...",
    "source": "人力资源管理制度.docx / 第四章 试用期管理"
  },
  "公司的组织架构类型是什么？": {
    "answer": "未在文档中找到相关信息",
```

### 来源格式规范

来源信息格式：`文件名 / 章节/页码`

示例：

- `员工手册.pdf / 第一章 公司概况`
- `人力资源管理制度.docx / 第四章 第一节 试用期`
- `劳动制度Nội quy lao động.pdf / 第2章 工作时间`
- `公司制度汇编.pdf / 第15页 薪酬管理`

## 问题库结构

问题库按模块和子类别组织，详见 `references/ec_questions.md`。

### 模块覆盖

1. **基本信息** - 公司概况、现有系统情况
2. **组织管理** - 组织架构设计、层级定义、变更审批
3. **组织信息** - 组织单元属性、状态管理
4. **岗位管理** - 岗位体系、编制管理
5. **人事管理** - 全生命周期管理（入职→转正→调动→离职）

## 自定义扩展

### 添加行业特定问题

编辑 `references/ec_questions.md` 文件，在"附录：行业特定问题"部分添加。

### 修改问题库

在 `scripts/generate_survey.py` 中修改 `DEFAULT_QUESTIONS` 字典。

### 临时问题调整（仅当前会话）

当用户在对话中需要调整问题时：
- Claude 会维护一个**临时问题列表**，存储在当前会话中
- 问卷生成使用这个临时列表，而非原始问题库
- 问题库文件不会被修改，确保下次对话时问题库完整

## 问题库管理规则

### 临时调整 vs 永久修改

| 场景 | 操作方式 | 影响范围 |
|------|---------|---------|
| 用户说"把这个问题改一下" | Claude在会话中维护临时问题列表 | 仅本次问卷 |
| 用户说"删除这个问题" | Claude从临时列表移除 | 仅本次问卷 |
| 用户说"添加一个新问题" | Claude添加到临时列表 | 仅本次问卷 |
| 需要永久修改问题库 | 手动编辑 `ec_questions.md` 或 `generate_survey.py` | 所有后续问卷 |

### Claude 的行为准则

1. **对话中的问题调整**：只在内存中维护临时列表，不修改文件
2. **生成问卷时**：使用用户确认后的临时问题列表
3. **问题库文件**：保持原样，作为后续对话的默认问题来源
