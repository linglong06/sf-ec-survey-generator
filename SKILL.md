# SF EC Survey Generator

生成 SAP SuccessFactors Employee Central 蓝图设计阶段调研问卷的专用工具。

## 工作流程

### 完整流程（推荐）

当用户提供制度文档时，按以下流程操作：

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  1. 展示问题    │ ──▶ │  2. 用户确认    │ ──▶ │  3. 提取答案    │ ──▶ │  4. 生成问卷    │
└─────────────────┘     └─────────────────┘     └─────────────────┘     └─────────────────┘
  展示完整问题列表        用户确认或修改问题      Claude分析文档          生成带答案的Excel
  按类别分组展示          确认后继续下一步        提取SF EC相关信息
                                                  记录答案来源
```

**Step 1: 展示问题列表（必须步骤）**

⚠️ **重要：在生成问卷前，必须先向用户展示完整问题列表**

Claude会按以下格式展示问题列表，让用户确认：

```
📋 SF EC 调研问卷问题库（共90题）

════════════════════════════════════════
1. 基本信息 (10题)
════════════════════════════════════════
  1.1 公司概况 (5题)
    1. 请简要介绍公司的业务范围和主营业务？
    2. 公司目前的员工总数是多少？
    ...

  1.2 现有系统 (5题)
    6. 目前使用的人事管理系统是什么？
    ...

════════════════════════════════════════
2. 组织管理 (12题)
════════════════════════════════════════
...

请确认：
1. 是否需要调整问题范围？（如只生成特定模块）
2. 是否需要修改或补充问题？
3. 确认无误后回复"继续"生成问卷。
```

**Step 2: 用户确认**

用户可以：

- ✅ **确认**：回复"继续"或"确认"，进入下一步
- ✏️ **修改**：指出需要修改的问题，Claude调整后再次确认
- 📦 **精简**：指定只生成特定模块（如"只生成组织管理和人事管理"）

**Step 3: 收集客户制度文档**

请用户提供以下文档（至少提供一种）：

- 公司制度汇编
- 员工手册
- 人力资源管理制度
- 组织架构说明文档
- 岗位职责说明书
- 薪酬管理制度
- 考勤管理制度
- 劳动合同管理规范

**Step 4: Claude从文档中提取答案**

Claude会阅读并分析文档，提取与SF EC模块相关的答案。

**重要规则：**

1. **答案必须来源于用户提供的文档**，严禁在文档之外自行编造或补充答案
2. **记录答案来源**：包括源文件名称和原文章节/段落
3. **找不到答案时**：在答案列填写"未在文档中找到相关信息"
4. **信息不完整时**：提取现有信息，标注"需补充"
5. **多个文档冲突时**：提取所有版本，标注"存在冲突，需确认"

关键信息提取方向：

- **组织管理**: 组织架构类型、层级定义、编码规则、审批流程
- **岗位管理**: 岗位体系、职级体系、编制管理规则
- **人事管理**: 入职流程、转正规则、调动流程、离职流程、合同管理
- **薪酬考勤**: 薪酬结构、工时制度、加班规则
- **数据权限**: 权限控制规则、敏感数据定义

**Step 5: 生成带答案的问卷**

```bash
python3 scripts/generate_survey.py -o ./问卷.xlsx -a answers.json
```

### 快速生成（无文档时）

如果没有客户制度文档，可直接生成空白问卷：

```bash
python3 scripts/generate_survey.py -o ./survey.xlsx
```

## 命令参数


| 参数                   | 说明                  | 默认值                   |
| -------------------- | ------------------- | --------------------- |
| `-o, --output`       | 输出文件路径              | `./sf_ec_survey.xlsx` |
| `-c, --categories`   | 指定类别（可多选）           | 全部类别                  |
| `-l, --language`     | 语言：`cn`/`en`/`both` | `both`                |
| `-a, --answers`      | 答案JSON文件路径          | -                     |
| `--no-answer-column` | 不包含答案列              | 包含                    |
| `--no-source-column` | 不包含来源列              | 包含                    |
| `--no-sub-category`  | 不包含子类别行             | 包含                    |
| `--template`         | 生成答案JSON模板          | -                     |
| `--list-categories`  | 列出所有类别              | -                     |


### 可用类别

- `基本信息` - 公司概况、现有系统
- `组织管理` - 组织架构、组织层级、组织审批
- `组织信息` - 组织基础信息、组织属性
- `岗位管理` - 岗位体系、岗位信息、岗位编制
- `人事管理` - 员工信息、入职、转正、调动、离职、合同、薪酬、考勤、权限、报表

## 使用示例

### 场景1：从制度文档生成问卷

用户提供员工手册后：

```
1. Claude展示问题列表，用户确认
2. Claude阅读员工手册
3. 提取与SF EC相关的答案，记录来源
4. 运行脚本生成带答案的问卷
```

```bash
python3 scripts/generate_survey.py -o ./客户A_EC调研问卷.xlsx -a answers.json
```

### 场景2：按模块生成

```bash
# 仅生成组织管理和岗位管理模块
python3 scripts/generate_survey.py -o ./survey.xlsx -c 组织管理 岗位管理
```

### 场景3：仅中文版本

```bash
python3 scripts/generate_survey.py -o ./survey.xlsx -l cn
```

## 输出格式

### 带答案的问卷（7列）


| 列   | 内容            |
| --- | ------------- |
| A   | 序号            |
| B   | 问题类别          |
| C   | 问题名称（中文）      |
| D   | 问题名称（英文）      |
| E   | 初步答案（来自制度文档）  |
| F   | 来源（文件名/章节）    |
| G   | 回答（空白，供确认/补充） |


### 空白问卷（5列）


| 列   | 内容       |
| --- | -------- |
| A   | 序号       |
| B   | 问题类别     |
| C   | 问题名称（中文） |
| D   | 问题名称（英文） |
| E   | 回答       |


## 答案JSON格式

```json
{
  "_说明": "这是答案模板文件",
  "_格式": "问题中文内容: {answer: 答案内容, source: 来源信息}",
  "公司目前的员工总数是多少？": {
    "answer": "约5000人",
    "source": "员工手册.pdf / 第一章 公司概况"
  },
  "试用期转正的业务规则是什么？": {
    "answer": "试用期一般为3-6个月，转正需通过部门考核...",
    "source": "人力资源管理制度.docx / 第四章 试用期管理"
  },
  "公司的组织架构类型是什么？": {
    "answer": "未在文档中找到相关信息",
    "source": ""
  },
  ...
}
```

### 来源格式规范

来源信息格式：`文件名 / 章节/页码`

示例：

- `员工手册.pdf / 第一章 公司概况`
- `人力资源管理制度.docx / 第四章 第一节 试用期`
- `劳动制度Nội quy lao động.pdf / 第2章 工作时间`
- `公司制度汇编.pdf / 第15页 薪酬管理`

## 问题库结构

问题库按模块和子类别组织，详见 `references/ec_questions.md`。

### 模块覆盖

1. **基本信息** - 公司概况、现有系统情况
2. **组织管理** - 组织架构设计、层级定义、变更审批
3. **组织信息** - 组织单元属性、状态管理
4. **岗位管理** - 岗位体系、编制管理
5. **人事管理** - 全生命周期管理（入职→转正→调动→离职）

## 自定义扩展

### 添加行业特定问题

编辑 `references/ec_questions.md` 文件，在"附录：行业特定问题"部分添加。

### 修改问题库

在 `scripts/generate_survey.py` 中修改 `DEFAULT_QUESTIONS` 字典。
